HTML Questions

    
1. What is difference between div and span in html

    Ans. Span and div are both generic HTML elements that group together related parts of a web page. However, they serve different functions.
    -->A div element is used for block-level organization and styling of page elements
    --> A span element is used for inline organization and styling.

    
2. What are the semantic elements in html?

    Ans.  Semantic HTML elements are those that clearly describe their meaning in a human- and machine-readable
    way. Elements such as header , footer and article are all considered semantic because they 
    accuratelydescribe the purpose of the element and the type of content that is inside them.

    
3. What is the difference between href attribute and src attribute ?

    Ans.  The href attribute specifies the location (URL) of a resource, that an anchor element points to, or the location of a linked resource, like a stylesheet.
    While the src attribute is used to embed a resource - usually URLs - into a document, think embedded images
    (img) and  tags

    
4. What is difference between xml and html?

    Ans. 
    HTML - HTML stands for Hypertext Markup Language. This computer language helps to make the data more interactive with multiple formatting features.
    XML - XML stands for eXtensible Markup Language and is used to transport and save data. The focus here is not on displaying or the appearance of the data.

CSS Questions

    
1.What is command for connecting local repository and github repository?

    Ans.
    git remote add origin (remote-repository-URL)
    git remote -v
    git push -u origin main

    
2. git command to change the directory of local folder?

    Ans.
    git mv old_directory new_directory
    git commit -m "Moved/renamed directory" (to commit changes)

    
3. git command to check the commits on the project?

    Ans.git log

    
4. git command for pushing all the code to ravi branch?

    Ans.
    git checkout ravi
    git add .
    git commit -m "Your commit message"
    git push origin ravi

    
5.Difference between the pull and push commands in git?

    Ans.git push: Uploads your local changes to a remote repository.
    git pull: Fetches and integrates changes from a remote repository into your local branch.

    
6. command tocheck the number of branches and command to switch the branch?

    Ans.Check the number of branches: git branch -a
    Switch to a different branch: git checkout (branch-name) or git switch (branch-name)

    
7. What are the stages of our project to reach to github repository?

    Ans.
    1. Initialize (if needed): git init
    2. Add Remote: git remote add origin (remote-repository-URL)
    3. Stage Changes: git add .
    4. Commit Changes: git commit -m "Your commit message"
    5. Push to GitHub: git push origin branch-name

    
8.The commands which we use in staging area?

    Ans.
    1. git add (file): Stage a specific file
    2. git add .: Stage all changes in the current directory and subdirectories
    3. git add -A: Stage all changes in the entire repository
    4. git status: View the status of changes in the staging area
    5. git reset (file): Unstage a specific file
    6. git reset: Unstage all changes in the staging area
    7. git rm --cached (file): Remove a file from the staging area
